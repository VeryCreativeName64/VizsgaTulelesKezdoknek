Most írásbelihez használható CSS jegyzetet kapsz, nagyon gyenge CSS tudásra optimalizálva, kifejezetten 2-es szint megcélzására.
Ez olyan, hogy egy feladatra szinte mindig rá tudod húzni, és reszponzivitás is benne lesz, mert az kritikus.
Nem lesz benne dizájnoskodás, csak túlélő CSS.

CSS JEGYZET – ÍRÁSBELIHEZ (MINIMUM TÚLÉLŐ SZINT)
1. Alap gondolkodás (ezt írd le fejben)
CSS célja:
* elrendezés
* olvashatóság
* reszponzivitás
Írásbelin nem szépség, hanem:
* ne csússzon szét
* mobilon se legyen használhatatlan

2. UNIVERSAL “MINDENRE JÓ” ALAP CSS
Ezt szinte mindig leírhatod, nem büntetnek érte.
1     * {
2       box-sizing: border-box;
3     }
4     
5     body {
6       margin: 0;
7       font-family: Arial, sans-serif;
8     }Mit csinál:
* box-sizing: border-box: nem borul fel a méretezés
* margin: 0: nincs felesleges fehér keret
* font-family: olvasható szöveg

3. KÖZÉPRE IGAZÍTÁS – FLEXBOX (MENTŐÖV)

Ha oszlop kell:

3       flex-direction: column;
4     }

8. RESZPONZIVITÁS – EZ KÖTELEZŐ!
Ha ez nincs, bukó lehet, szóval EZT MEG KELL ÍRNI.
Media query alapminta
1     @media (max-width: 768px) {
2       .container {
3         flex-direction: column;
4       }
5     
6       table {
7         font-size: 14px;
8       }
9     }Mit jelent:
* 768px alatt (mobil, tablet)
* oszlopba vált
* kisebb betűméret
Vizsgán elég annyit mondani:
„Mobil nézethez alkalmazkodik.”

Reszponzivitas reszletesebben
Nagyon jó kérdés – itt szokott mindenki elcsúszni, mert a reszponzivitás nem csak egy media query, hanem át kell gondolni, hogyan viselkedjenek az elemek kisebb képernyőn.
 Most lassan, lépésről lépésre, írásbeli-logikával magyarázom el.
Nem dizájn, hanem túlélési stratégia.

1. Mit jelent VALÓJÁBAN a reszponzivitás?
Írásbelin ezt jelenti:
„Az oldal kisebb képernyőn is használható marad:
 – nem lóg le
 – nem túl kicsi
 – az elemek egymás alá kerülnek”
Tehát nem új oldal, csak:
* más elrendezés
* más méretek
* kevesebb „egymás mellett”

2. AZ ALAP ELV (ezt jegyezd meg!)
Desktopon:
* sok elem egymás mellett
 Mobilon:
* ugyanazok az elemek egymás alatt
Ez a kulcsgondolat.

3. HOGYAN IGAZÍTOD HOZZÁ A TÖBBI ELEMET?
3.1. Először: legyen egy „keret”
Mindig legyen egy fő wrapper:
.container {
  max-width: 1200px;
  margin: 0 auto;
  padding: 10px;
}
Mit csinál:
* nem nyúlik szét nagy képernyőn
* mobilon is van tér
Ez reszponzivitás-alap.

4. FLEXBOX + RESZPONZIVITÁS (A LEGFONTOSABB RÉSZ)
Desktop (alap állapot)
.list {
  display: flex;
  gap: 10px;
}
Ez:
* egymás mellé rakja az elemeket

Mobil (media query-ben)
@media (max-width: 768px) {
  .list {
    flex-direction: column;
  }
}
Mit történik?
* ugyanaz a lista
* csak egymás alá kerül
Ez a reszponzivitás lényege.

5. TÁBLÁZATOK RESZPONZÍV KEZELÉSE (KRITIKUS!)
Táblázat mobilon mindig problémás.
Alap tábla:
table {
  width: 100%;
  border-collapse: collapse;
}
Mobilra „mentőmegoldás”:
@media (max-width: 768px) {
  table {
    font-size: 14px;
  }
}
Ez:
* kisebb betű
* nem lóg ki
Írásbelin ez már elfogadható.

6. FORMOK RESZPONZIVITÁSA (GYAKORI FELADAT!)
Alap:
form {
  max-width: 400px;
  margin: 0 auto;
}
Inputok:
input, select, button {input,
  width: 100%;
Mit csinál:
* mobilon nem kell zoomolni
* könnyű kattintani
Ez UX + reszponzivitás együtt.

7. TIPIKUS PROBLÉMA: „SZÉTCSÚSZIK MOBILON”
