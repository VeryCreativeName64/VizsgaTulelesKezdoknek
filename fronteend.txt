5. PAGE vs COMPONENT SZÉTVÁLASZTÁS (KRITIKUS!)
Page (pages/)
* route-hoz kötött
* adatot tölt
* state-et kezel
Component (components/)
* újrahasznosítható
* csak props
* nincs API hívás
Fejlesztési sorrend:
1. létrehozod a Page-et
2. Page-be beteszed a Componenteket

6. ALAP PAGE SKELETON (EZT MEG KELL JEGYEZNI)
Minden Page így indul:

2     const [loading, setLoading] = useState(true);
3     const [error, setError] = useState("");
4     
5     useEffect(() => {
6       loadData();
7     }, []);Miért?
* loading állapot
* hiba kezelése
* aszinkron adat
Ez vizsga-biztos minta.

7. SERVICE RÉTEG LÉTREHOZÁSA (API HÍVÁSOK)
services/api.js
1     import axios from "axios";
2     
3     const api = axios.create({
4       baseURL: "http://localhost/api"
5     });
6     
7     export default api;Miért külön?
* Page ne legyen „szemetes”
* könnyebb tesztelni
* átláthatóbb

Konkrét service fájl
1     export const fetchItems = async () => {
2       const res = await api.get("/items");
3       return res.data;
4     };Szabály:
* service csak HTTP
* nincs state
* nincs navigate

8. PAGE + SERVICE ÖSSZEKÖTÉSE
Page-ben:
1     const loadData = async () => {
2       try {
3         setLoading(true);
4         const result = await fetchItems();
5         setData(result);
6       } catch (e) {
7         setError("Hiba történt");
8       } finally {
9         setLoading(false);
10       }
11     };Ez:
* tiszta
* érthető
* vizsgán nagyon jó
